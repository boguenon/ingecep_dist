Database Configuration

Connection Settings:

Setting up INGECEP! to run on your server need to complete this setting.

This step will guide your for easy database configuration. Choose your preferred settings:

MetaDB Settings

Select the type of database from the drop down list. This will generally be MySQL_5.

Enter the hostname of the database server Orbit! will be installed on. This may not necessarily be the same as your Web server so check with your hosting provider if you are not sure.

Enter username, password and database name you wish to use with Orbit!. These must already exist for the database you are going to use.

R-OLAP DB Settings

Add or modify your database assets to analyze with Orbit!

Click database poolname on the following list to modify connection parameter.

    Settings
    This is database connection poolname to use on Orbit system. This shall be an alias and recognizable name for your distinguishing database system.

    This is probably MySQL_5

    This can be the default MySQL username root, a username provided by your hosting provider, or one that you created in setting up your database server.

    Using a password for the MySQL account is mandatory for site security. This is the same password used to access your database. This may be predefined by your hosting provider.


        

    Advanced Settings
    By selecting database type on the basic settings, this will automatically changed with suggested Java class for connecting your database system.

    By selecting database type on the basic settings, this will automatically changed with suggested Java class for connecting your database system. Modify and configure detailed information to connect your database system.

    Optional! Schema name is database name on MySQL. Analytical ROLAP service, SQL query is generated with this schemaname.

    For consistency check on database connection with pool, this SQL query is used to validate the connection. If failed with this query then the connection is removed and new connection is generated automatically.

    To protect system from large data query, this limit the maximum rows fetched from AdHoc generated query.
